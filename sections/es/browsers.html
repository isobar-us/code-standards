Pruebas y Soporte de Navegadores

La audiencia de hoy puede elegir entre toda una gran cantidad de navegadores web, cada uno con una experiencia ligeramente (o dramáticamente) diferente. Como desarrolladores, es nuestra responsabilidad elegir cómo se muestran a los usuarios las páginas web que construimos. En esta sección se describe cómo nosotros, en Isobar, tomamos algunas de estas decisiones clave.

Que soportamos

Isobar soporta cualquier navegador con grado A en el Soporte Graduado de Navegadores de Yahoo, con la excepción de Opera. Puede haber excepciones, teniendo en cuenta los mercados regionales y sus indicadores particulares.

Haremos todo lo posible para soportar cualquier otro navegador de misión crítica exigido por el cliente (Internet Explorer 5.5, Opera, Konqueror, Safari 3 para PC, etc), aunque no podemos garantizar que toda la funcionalidad esté disponible.

Como realizamos las pruebas

Las realización de pruebas integrales de navegador son imprescindibles en cada proyecto web de Isobar. Debe hacerse un esfuerzo considerable para hacer las pruebas entre múltiples navegadores y plataformas a fin de garantizar la calidad y una experiencia de usuario consistente. La creación de un entorno de pruebas puede ser un desafío, pero bien vale la pena hacerlo.

En Microsoft Windows

Pruebas de IE

Dado que es imposible tener más de una sola copia de Microsoft Internet Explorer instalada en un PC, las pruebas de IE son un reto. Afortunadamente, Microsoft finalmente ha publicado versiones de desarrollo de Internet Explorer antiguos disponibles para su descarga. Estos discos duros virtuales (VHD por sus siglas en inglés) ejecutan versiones simplificadas de Microsoft Windows que expiran en el tiempo. Normalmente hay que reconfigurarlas luego de algunos meses. Las copias de desarrollo de Microsoft Windows de su licencia de MSDN (si están disponibles) también pueden ser una opción, dependiendo de a que tiene acceso.

  • Virtual PC - Virtual PC debe estar instalado en el equipo, y si está en Windows 7, debe utilizar "Modo XP".
  • Imágenes VPC de Microsoft Windows - Hay disponibles diferentes variedades de imágenes de disco duro virtual. Puede que necesite instalar varias para tener una suite de pruebas integral, dependiendo de su proyecto.

Además, hay otras opciones menos eficaces para pruebas de IE (normalmente no son recomendadas) que incluyen: IETester, el cual es todavía mejor que Multiple_IE e IE7 standalone.

Firefox
  • Firefox 3.6+ debe instalarse de forma nativa y la versión 3.0 está disponible como versión portátil.
  • Si está preparado para ello, instale Firefox 3, 3.5 y/o 3,6 paralelamente a FF4. El Administrador de Perfiles de Firefox le permite instalar en directorios diferentes y mantener perfiles diferentes para cada uno.
Safari para PC
  • Utilice la última versión de Safari para PC. Es 98% compatible con Safari en Mac OS X, pero no completamente, así que pruebe allí si es una plataforma necesaria.
Opera
  • Puede descargar versiones antiguas desde el archivo de Opera. Instale en carpetas diferentes para ejecutar las diferentes versiones
Google Chrome y Versiones de Chrome

Google Chrome se actualiza contínuamente y la mayoría de todos los usuarios tienen a menudo la versión más reciente. Ojalá fuera así de fácil con todos los otros navegadores. No hay de que preocuparse con navegadores antíguos basados en Google Chrome.

En Mac OS X

Los principales navegadores de Mac OS X, Mozilla Firefox, Google Chrome y Safari de Apple ofrecen experiencias de navegador prácticamente idénticas en comparación con sus contrapartes de Windows. Dicho esto, algunas diferencias a nivel de sistema operativo pueden estar presentes y los sitios web deben probarse en ambas plataformas. Las diferencias típicas giran alrededor de como se muestran las fuentes y por lo tanto surgen a veces problemas de espaciado.

Pruebas de Windows en Mac

Hay una serie de opciones para las pruebas de navegadores basados en Windows en Mac OS. Primero, los equipos Mac ofrecen una partición "boot camp" que le permitirá arrancar el Mac en una partición de Windows de Microsoft. Se trata de un entorno de prueba complejo pero completo. Una vez haya arrancado Windows, las opciones normales de Windows estarán presentes.

Otras opciones incluyen la virtualización de Windows dentro de Mac OS X, que le permite literalmente ejecutar Windows dentro de Mac OS.

Los Discos Duros Virtuales (VHD) de Microsoft se pueden abrir o convertir en la mayor parte de estas opciones, permitiendo, por lo tanto, un proceso de pruebas al mismo grado que los usuarios de Windows tienen a su disposición. Aunque, ya que puede también hacer las pruebas en el Mac simultáneamente, algunos podrían argumentar que tiene mayor flexibilidad…

  • Parallels - Está disponible y puede que ya haya sido instalado por el departamento de TI de Isobar en su Mac.
  • VMWare Fusion - VMware Fusion ofrece el mismo nivel de virtualización de Windows a través de su producto Fusion.
Mozilla Firefox

Al igual que en Windows, puede instalar y ejecutar múltiples copias de Mozilla Firefox en su Mac, aunque es un poco más complicado el configurar múltiples perfiles mediante el Administrador de Perfiles. Dicho esto, hay algunos trucos que puede utilizar con Automator para crear diferentes perfiles y ejecutarlos agradablemente.

Errores en versiones independientes de IE

Nota: El IE6 independiente (standalone) tiene una implementación con errores de la transparencia en algunos casos. Esto resultará en que cualquier transparencia aplicada con un filtro CSS, como una transparencia alfa o un PNG de 24 bits, fallará. En caso de que deba probar transparencias, necesitará una instalación nativa de IE6.

Se ha observado que el IE7 usando la plataforma de Windows Vista tiene diferencias con respecto a IE7 en Windows XP, por lo tanto, es posible que desee asegurarse de que alguien en su equipo tiene esta configuración también. Se sabe que IETester repara un número de estos problemas, al igual que los navegadores Xenocode.

Resolución del Navegador

Además de tener que considerar a los navegadores, los desarrolladores deben estar conscientes de las resoluciones de pantalla de su audiencia. Mientras las pantallas de los monitores se hacen más grandes, la amplitud de las resoluciones también crece. A continuación se presentan algunas pautas que le ayudarán en el trabajo con las resoluciones.

Resolución 1024px

  • El doblez se estima en 570px.
  • Anchura óptima: 960px - Tiene un padding cómodo a ambos lados, es divisible por muchos números y también juega bien con los anchos estándar para anuncios publicitarios de la IAB
  • Mayor anchura: 970px - Todavía tiene algo de padding a ambos lados en la mayoría de los navegadores. La matemática juega bien con la Proporción áurea
  • Anchura máxima: 996px - Sin incurrir en barras de desplazamiento horizontales en los principales navegadores. Basándose en esta investigación el máximo es 1003 px de ancho si no quiere una barra de desplazamiento horizontal.
