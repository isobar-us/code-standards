Accesibilidad

Este es un tema bastante amplio que es aparte de todo lo que entra en un sitio web. La accesibilidad tiene que ver con todo, desde cómo se escribe el contenido hasta la estructura del etiquetado HTML, la apariencia y estilo, el mejoramiento progresivo y la degradación agradable, además de cuán semántico y lleno de significado es su código cuando es leido por los navegadores.

Nosotros unimos personas y marcas como nunca antes. Es importante que lleguemos a la mayor cantidad posible de personas para proporcionarle experiencias nuevas y emocionantes "como nunca antes". Algunas áreas a considerar para mejorar la accesibilidad durante el desarrollo de cualquier proyecto son:

  • Etiquetado y semántica
  • Usabilidad de la interfaz, a través del teclado y con JS desactivado
  • Uso correcto de las etiquetas, atributos y descripciones
  • Contenido de texto para todo el contenido que no sea texto
  • Contenido de resumen para lectores de pantalla

Estas áreas están todas estrechamente relacionadas, por lo que puede ser muy fácil romperlas dependiendo del tipo de proyecto. Aquí tiene algunas cosas que deben hacerse para lograr una página web accesible:

Saltar la Navegación

Si el sitio tiene elementos de navegación antes del contenido principal, proporcione un enlace de "Ir al contenido principal" para los lectores de pantalla. Este es un enlace que normalmente se oculta mediante CSS, pero estará disponible para un lector de pantalla a fin de saltar la navegación e ir directamente al contenido. A veces puede ser útil incluir un enlace de saltar al pie de página o al mapa del sitio.

Índice de Tabulación

El índice de tabulación de un documento se inicia con el primer enlace en el documento y funciona a través de todos ellos en el orden del código hasta el final. Esto es a menudo inútil y contra-intuitivo a como aparecen los elementos en pantalla. Así que asegúrese de que el orden de tabulación de los enlaces y elementos activos sea lógico usando el atributo tabindex. Debe poder tabularse a través de los elementos de formulario en el orden en que un usuario espera completarlo, no necesariamente en el orden de las etiquetas en el código HTML.

Acrónimos/Abreviaturas

Si su contenido utiliza abreviaturas o acrónimos, éstos deben ser expandidos mediante la etiqueta abbr

Imágenes.

Todas las imágenes deben tener un atributo alt. Los atributos alt deben tener un significado adecuado y describir el contenido de la imagen. Si la imagen es puramente decorativa, los atributos alt deben marcarse como vacíos utilizando alt="". No coloque algo como alt="Esta imagen es sólamente de decoración" ya que el lector de pantalla lo leerá y es realmente molesto.

No utilice imágenes de espaciado u otros efectos similares. Esto debe hacerse mediante CSS.

Flash y Reemplazo de Imágenes

Siempre utilice contenido HTML de reserva para Flash. Todas las imágenes promocionales deben utilizar reemplazo de imágenes CSS en lugar de etiquetas alt.


Tener una reserva para Flash e imágenes también mejorará el SEO.

Tablas

Las tablas deben utilizarse exclusivamente para datos tabulares. No deben utilizarse para la presentación o con fines de diseño. Cuando utilice tablas:

  • Utilice el elemento th para los encabezados de la tabla. Si los encabezados están abreviados, utilice el atributo abbr para expandirlas.
  • Para tablas complejas, utilice el atributo scope en los encabezados de tabla para indicar a que se aplica.

Color

El uso del color debe cumplir lo siguiente:

  • El texto debe ser legible, con alto contraste. Utilice un fondo claro con texto oscuro o viceversa. El texto muy claro sobre un fondo claro es prácticamente imposible de leer para la mayoría de las personas y puede ser aún peor para las personas con daltonismo.
  • No utilice rojo y verde juntos para hacer distinciones entre funcionalidades críticas - un pequeño porcentaje de las personas no serán capaces de percibir tal distinción.

El sitio de la WAI contiene una lista de herramientas sobre daltonismo que le permiten ver sitios tal como lo verían las personas con diferentes tipos de daltonismo.

JavaScript

Las páginas todavía deben ser funcionales con JavaScript desactivado. Las aplicaciones deben degradar agradablemente, o cuando sea posible, hacer uso de polyfills compatibles con navegadores para agregar funcionalidades a los navegadores antiguos a fin de habilitar características que pueden no ser soportadas.

Sección 508

Las interfaces desarrolladas por Isobar deben cumplir con las normas de la sección 508.
